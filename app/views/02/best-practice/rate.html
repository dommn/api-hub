Rate limiting

Why it's important

Rate limiting prevents abuse of your API by restricting the number of requests a client can make within a specific time frame. It helps maintain API performance, protects against denial-of-service attacks, and ensures fair usage among all users.

What it means

  • Set clear limits on the number of requests a user can make in a given period (e.g., 100 requests per minute).
  • Provide a way to track the usage of rate limits with response headers (e.g., X-Rate-Limit).
  • Return an appropriate HTTP status code (429 Too Many Requests) when the rate limit is exceeded.
