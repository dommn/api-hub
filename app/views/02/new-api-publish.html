Publish and support your API

Once your API is ready, you should publish it and make sure it can be found, accessed and supported over time.

Why this matters

Publishing your API allows others to find and use it. Good support and monitoring help ensure it remains reliable and trusted.

What to do

  • List your API in the GOV.UK API Catalogue
  • Include a short, clear description of what the API does
  • Make sure the listing includes documentation and support contact details
  • Set up monitoring for availability, performance and error rates
  • Track how the API is being used and by whom

Set up support

  • Provide a support email address or ticketing system
  • Monitor for common issues or frequently asked questions
  • Be clear about support hours and service level expectations
  • Publish changes and fixes in a changelog or update log
Make it easy for users to ask questions, request access, or report bugs.
