API glossary of terms

Common terms used when working with APIs, explained in plain English.

API (Application Programming Interface)
A way for two systems to talk to each other automatically.
Endpoint
A specific URL where a service or function can be accessed (e.g. /check-status).
Request
A message sent to the API asking it to do something or return information.
Response
What the API sends back after receiving a request â€” often in JSON format.
Authentication
How an API checks the identity of the person or system making the request.
Token
A secure piece of information used to prove identity to an API.
JSON (JavaScript Object Notation)
A lightweight format for sending data, used by most APIs.
REST
A common API design approach using standard web methods like GET and POST.
Rate limiting
A restriction on how many times you can call an API in a given time period.
Versioning
Labeling different versions of an API so users can manage changes over time.
SDK (Software Development Kit)
A toolkit provided to help developers use an API more easily.
Sandbox
A test version of the API that doesn't affect real data or users.
