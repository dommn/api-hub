How to get started

If you’re new to APIs, or planning to build or use one in your service, this guide will help you take the right steps from the start.

1. Understand the problem you're solving

Start with the user need or business problem. Ask:

  • What task needs to happen faster, more reliably, or automatically?
  • Who will use the API — internal teams, the public, partners?
  • Is it about sharing data, triggering actions, or retrieving information?

An API is only useful if it helps people complete a task or improves a process.

2. Check if something already exists

Before building anything, check what’s already available:

  • Search the GOV.UK API catalogue
  • Talk to other departments or your organisation’s tech leads
  • Look for APIs you can reuse or extend

Reusing existing APIs saves time and avoids duplication.

3. Read the guidance

Familiarise yourself with government standards:

  • API design and delivery guidance
  • Security and authentication
  • Data protection and privacy

4. Talk to the right people

Involve others early — including:

  • Policy and service owners
  • Security and data protection teams
  • Developers and technical architects
  • Other departments that might use or maintain the API

If unsure, contact the API Hub team for support or to connect with others.

5. Start small, build iteratively

It’s better to build a simple, well-scoped API that solves a clear problem than to start with a large, complex one.

  • Define a single endpoint and test it
  • Use real-world data and users in testing
  • Get feedback and improve before scaling

6. Document and share

Good APIs are discoverable and easy to use. Make sure to:

  • Write clear, plain English documentation
  • Include example requests and responses
  • Explain how to authenticate and get access
  • Link to your service or support team

You can publish your API in the GOV.UK API catalogue to help others find and reuse it.
