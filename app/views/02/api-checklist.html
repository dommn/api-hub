API development checklist

Understand the need

  • I know what the API is for and who will use it
  • The problem the API solves is clear
  • I’ve spoken to users, service teams or data owners

Check what already exists

  • I searched the GOV.UK API catalogue
  • I’ve checked for similar APIs in my organisation
  • I’ve avoided duplicating existing functionality

Plan and design the API

  • I’ve defined clear endpoints, inputs and outputs
  • I’m following REST and JSON standards (or have a good reason not to)
  • I’ve considered security, authentication and error handling
  • I’ve used common design patterns where relevant

Build and test

  • I’ve built the API using modern, secure tooling
  • I’ve created test environments (sandbox, staging)
  • I’ve written automated tests for key behaviour
  • I’ve included versioning and monitoring

Document and share

  • I’ve written plain English documentation
  • I’ve included request/response examples and authentication steps
  • I’ve published or plan to publish in the GOV.UK API catalogue
  • I’ve included support and contact details

Maintain and improve

  • I’ve set up logging, metrics and error alerts
  • I know how I’ll manage updates and deprecations
  • I’ve collected feedback or usability testing results
  • The API has a clear owner and support process
