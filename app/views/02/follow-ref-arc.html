Follow government reference architecture

The Government API Reference Architecture outlines best practice for building consistent, secure and scalable APIs.

Key principles include:

  • RESTful design using standard HTTP methods and resource-based URLs
  • Standardised formats, such as JSON for payloads and ISO 8601 for dates
  • Consistent error handling, including clear status codes and structured error messages
  • Token-based authentication using OAuth 2.0 or similar standards
  • Separation of concerns, such as API gateways or backend-for-frontend layers

Using this architecture helps other teams understand and reuse your API with less friction.
