Plan your API design

Good API design is consistent, predictable and easy to use. It saves developers time and reduces the need for support.

Why this matters

Well-designed APIs are more likely to be reused and integrated correctly. They reduce complexity and technical debt.

What to do

  • Use RESTful design principles unless thereâ€™s a good reason not to.
  • Structure URLs around real-world resources, for example /applications/{id}.
  • Keep parameters and responses simple, flat, and predictable.
  • Use consistent naming and pluralisation for resources.
  • Return standard HTTP status codes and clear error messages.
  • Default to using JSON as the response format.

Useful patterns

  • Pagination and filtering (for lists)
  • Metadata in responses (such as total, offset, limit)
  • Versioning your API in the URL path (for example /v1/)

Refer to the common API design patterns for guidance.
