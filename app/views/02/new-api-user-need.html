Understand the user need

Every API should solve a real problem. Don’t start with the technology – start with the need.

Why this matters

APIs are useful only when they help users achieve a task. Understanding the need ensures your API has value and purpose.

What to do

  • Identify the users of the API: internal developers, partners, suppliers, or the public.
  • Define the task they are trying to complete with the API.
  • Map the user journey or workflow the API will support.
  • Consider if the API handles sensitive or personal data.

Questions to ask

  • Who needs this API, and why?
  • What problem does it solve?
  • What would happen if the API didn’t exist?
  • Is an API the right tool, or is another solution better?
Talk to service teams, delivery managers, and potential API users before writing any code.
