What you need to know

You don’t need to be a developer to understand APIs — but here are some key terms:

  • Endpoint – the address where an API is accessed (like a URL)
  • Request – when your service asks for data from an API
  • Response – what the API sends back
  • Authentication – how an API checks who you are
  • Documentation – guidance on how to use the API

Most APIs use formats like JSON and follow standard design approaches such as REST.
