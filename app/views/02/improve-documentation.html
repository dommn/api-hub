Improve your documentation

Clear documentation reduces support and helps developers get started faster. Include:

  • Plain English overview of what the API does
  • Authentication instructions
  • Example requests and responses
  • Full schema or downloadable OpenAPI spec
  • Known errors and how to handle them
  • Contact details and support options

Host documentation on a public developer portal and link to it from your API listing in the catalogue.
