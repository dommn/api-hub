Design your API

API discovery

Before you commit to designing a new API, you need to understand the problem that needs to be solved.

That means learning about:

  • who are your users and what are their user needs
  • what users need from your API to achieve their goal by doing user research

This part of your project is called the discovery phase.

For an API, the user is usually a developer who wants to consume your API to deliver a service. The developer will have needs based on:

  • the service they are developing
  • how easy it is to write code to consume your API
