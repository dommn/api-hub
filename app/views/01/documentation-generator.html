OpenAPI Documentation Generator

Overview

The OpenAPI Documentation Generator is a Java-based tool that processes your OpenAPI YAML specification and generates enriched, accessible HTML documentation. It adheres to Government Digital Service (GDS) standards and includes the following features:

  • Customizable input and output file paths.
  • Null-safe printing that replaces missing values with “N/A”.
  • Detailed, step-by-step logging using SLF4J.
  • Built-in accessibility enhancements in the generated HTML.

Prerequisites

Before using this tool, ensure you have:

  • Java 17 (or later) installed.
  • Maven installed.
  • An OpenAPI YAML file containing your API specification.

Usage

Run the JAR file:

Execute the JAR file from the command line with your desired options:

java -jar cddo-openapi-documentation-generator-0.0.1-SNAPSHOT.jar [options]

Command-Line Options:

  • --help or -h
  • Displays a help message with usage instructions.
  • --openapi=
  • Specifies the path to the OpenAPI YAML file.
  • Examples: o --openapi=classpath:openapi.yaml, o --openapi=file:/path/to/openapi.yaml
  • --output=
  • Specifies the output file path for the generated HTML documentation.
  • Example: o --output=/path/to/generated-doc.html

If no arguments are provided, the tool defaults to using classpath:openapi.yaml for input and generates generated-doc.html in the current directory.

Logging

Monitor logging:

The application uses SLF4J for logging and provides clear, step-by-step progress updates during execution. Key logged steps include:

  • Startup: The generator starts and initializes.
  • File Loading: The OpenAPI YAML file is loaded from the specified path.
  • Parsing: The YAML file is parsed into an OpenAPI object.
  • HTML Generation: The HTML documentation is generated based on the OpenAPI specification.
  • Output: The generated HTML is written to the specified output file.

Monitor the console for detailed logging information during these processes.

Accessibility Enhancements

Review Accessibility Enhancements:

The generated HTML documentation includes the following accessibility features:

  • A “Skip to main content” link is provided at the top for keyboard users.
  • Proper use of <header>, <main>, and <footer> elements with appropriate ARIA roles ensures that assistive technologies can navigate the page easily.
  • The SVG logos include descriptive <title> tags and ARIA labels to provide context for screen readers.
